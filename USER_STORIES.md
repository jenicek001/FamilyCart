# Project Requirements Document: The Family Cart

## User Stories and Functional Requirements (MVP)

| Requirement ID | Feature | User Story | Functional Requirement | Expected Behavior/Outcome |
|---|---|---|---|---|
| FR001 | Email/Password Auth | As a user, I want to have a simple sign-up / sign-in process for login to the app - log in with email/password, so I can access my shopping lists securely. | The system shall provide email/password registration and login, with secure session management using JWT. | Users can create accounts and log in / log out. |
| FR002 | OAuth Login | As a user, I want to log in with Google or Apple, so I can use my existing accounts. | The system shall support OAuth 2.0 login with Google and Apple. | Users can authenticate using Google or Apple and access their shopping lists. |
| FR003 | Create List | As a user, I want to create new shopping lists, so I can organize my shopping. | The system shall allow users to create new, empty shopping lists. | Users can create and view new shopping lists. |
| FR004 | View Lists | As a user, I want to view all my own and shared shopping lists, so I can manage them easily. | The system shall display all shopping lists owned by the user and all shopping lists shared with the user. | Users see a list of their shopping lists. |
| FR005 | Rename List | As a user, I want to rename my owned shopping lists, so I can keep my lists organized. | The system shall allow users to rename their shopping lists. | Users can update the name of any list they own. |
| FR006 | Search & Add Item | As a user, I want to add items to a shopping list by simply typing part of item name. It shall use my shopping history or global items list to search for items as I type, so I can plan my shopping quickly and efficiently. | The system shall allow users to add items to the active list, and provide unified, as-you-type search for items from history, common items, and the current list. Matching Items shall appear immediately as a list of Items to allow me to choose from. | Items are added to the selected list with all specified details, and search results update in real time as the user types. |
| FR007 | LLM Integration | As a user, I want the application to use AI for item classification, translation, and icon generation, so I can have a smarter shopping experience. | The system shall integrate with LLM APIs for item classification, translation, and icon generation. | Items are classified, translated, and have icons generated automatically using LLM APIs. |
| FR008 | Sorting Items based on Category | As a user, I want to see items sorted by category, so I can find them easily in the list. | The system shall allow sorting items in a list by category. | Items are displayed in the list grouped and sorted by their categories. |
| FR009 | Quantify Item | As a user, I want to specify the quantity of items in various units, so I can accurately plan my shopping. | The system shall allow users to specify item quantities in pieces, grams, liters/milliliters, or local unit alternatives. | Users can add items with specified quantities in their preferred units. |
| FR010 | Item Details | As a user, I want to add notes to items, so I can remember specific details about them. | The system shall allow users to add optional notes to each item. | Notes are saved with each item and displayed in the list. |
| FR011 | Item Status | As a user, in a shop, I want to mark items as purchased or currently unavailable, so my family knows the shopping status. | The system shall allow marking items as purchased or temporarily unavailable. | Item status is updated and visible to all list members. Items marked as purchased or currently unavailable are clearly indicated and are moved to the end of the list. |
| FR012 | Item Context | As a user, I want to see who requested or last updated an item, so I know who is responsible. | The system shall display the nickname and last action (with date/time) for each item. | Item details show requester and last updater info. |
| FR013 | Edit Item | As a user, I want to edit item details, so I can correct mistakes or update information. | The system shall allow editing of item details (name, quantity, notes, etc.). | Users can update any field of an item in their or shared lists. |
| FR014 | Remove Item | As a user, I want to remove items from a list, so I can keep my or shared list relevant. | The system shall allow users to remove items from a list (soft delete). | Removing items from the list requires user confirmation. Items are removed from the list but not deleted from the database - they stay in history. |
| FR015 | Reorder Items | As a user, I want to reorder items in a shopping list, so I can prioritize my shopping. | The system shall allow users to reorder items in a list. | Users can drag and drop items to change their order in the list. |
| FR015a | Reorder Items within Category | As a user, I want to reorder items within their category in a shopping list, so I can organize my shopping within specific food groups. | The system shall allow users to reorder items within their assigned category. Changes shall be synchronized in real-time to all shared users. | Users can drag and drop items to change their order within a category. The new order is immediately visible to all members sharing the list. |
| FR015b | Reorder Categories | As a user, I want to reorder entire categories of items in a shopping list, so I can arrange the list according to my shopping route or preference. | The system shall allow users to reorder entire categories (along with all items in them). Changes shall be synchronized in real-time to all shared users. | Users can drag and drop categories to change their order in the list. The new order of categories and their items is immediately visible to all members sharing the list. |
| FR016 | Delete List | As a user, I want to delete entire shopping lists, so I can manage my lists. | The system shall allow users to delete their shopping lists. | Lists are deleted after confirmation. |
| FR017 | Invite Members | As a user, I want to invite family members to a shopping list, so we can collaborate. | The system shall allow inviting other registered users to a list by email/username. | Invited users gain access to the shared list after their acceptance. |
| FR018 | Real-time Updates | As a user, I want real-time updates on shared lists, so everyone sees changes instantly. | The system shall provide real-time updates via WebSockets for shared lists. | All members see changes to lists and items in real time. |
| FR019 | Manage Members | As a user, I want to view and manage members of a shared list, so I can control access. | The system shall display list members and allow removing members. | Users can see and manage who has access to each list. |
| FR020 | Responsive UI, Modern UX | As a user, I want the application to work on mobile, tablet, and desktop, so I can use it anywhere. | The system shall provide a responsive UI that adapts to different screen sizes. System shall use modern frameworks to provide a modern look - e.g. Tailwind CSS | Users can access and use the application on any device. It will provide a beautiful modern UX |
| FR021 | I18n Support | As a user, I want the application to support multiple languages, so I can use it in my preferred language. | The system shall support internationalization (I18n) for item categories and UI components. | Users can switch languages, and item categories are translated via LLM API calls. |
| FR022 | Performance Goals | As a user, I want the application to be fast and responsive, so I can use it efficiently. | The system shall meet performance goals: search under 200ms, updates under 1s. | Users experience minimal latency during searches and updates. |
| FR023 | Limit item searching to recently used items | As a user, I want to keep searching for items (to be added into list) fast and efficient, so only recent items (e.g. last 6 months or 1 year) are offered in search results | The system shall use for each item last-updated attribute and a defined retention period to determine if an item should be included in search results. Items not updated within the retention period shall be excluded from search results. | Users can only see and search for recently used items, improving search efficiency and relevance. |
| FR024 | Shopping History | As a user, I want to see my shopping history, so I can track what I have purchased in the past. | The system shall maintain a shopping history for each user, including item details, purchase dates, and quantities. | Users can view their shopping history and re-add items to their lists easily. |
| FR025 | Recommended Items | As a user, I want to see recommended items based on my shopping history, not to forget items, which I forgot to add to the list once they gone out of home stock. | The system shall display May you also need section between Items to buy and Checked/Purchased items, showing recommended items based on the user's shopping history and frequently purchased items. | Users can quickly add recommended items to their shopping list with a single click. |
| FR026 | SmartSearchBar | As a user, I want to have a SmartSearchBar that allows me to type item name or their parts and it will immediately as I type show me matching items from my shopping history, common items, and the current list. | The system shall provide a SmartSearchBar that searches for items as the user types, showing matching items from history, common items, and the current list. | Users can quickly find and add items to their list without needing to type the full name. |
